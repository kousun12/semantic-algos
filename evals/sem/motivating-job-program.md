# Evaluation: motivating job program

## Purpose

Exercise the intended top-level experience: compile one emotionally messy
natural-language request into shared upstream understanding and three isolated
literary/comic branches, while translating named artists into broad mechanisms
instead of asking workers for imitation.

## User-style invocation

Present only this block to the system under evaluation:

```text
$sem-run thinking about leaving my job, it's a drag, i get paid a lot and am respected but it is soul crushing - tell me a story like kafka would tell it, then a song like bob dylan would write it, and finally a joke like norm macdonald would tell it
```

## Compilation must preserve

- The dilemma retains both sides: high pay and respect versus drudgery and the
  felt loss of soul. It must not silently become advice to quit or stay.
- Shared upstream work identifies or sharpens one usable central tension before
  the terminal forms fan out. The compiler may use repository skills or a
  clearly defined local tension extractor, but every semantic application must
  remain identifiable.
- The terminal operations resolve to the actual `parable`, `lyric`, and `joke`
  contracts where those contracts fit the requested story, song, and joke.
- “Kafka,” “Bob Dylan,” and “Norm Macdonald” remain in the preserved request and
  compile notes as provenance, but executable configuration lowers each named
  reference to broad mechanisms rather than imitation or borrowed voice.
- Suitable mechanism families include bureaucratic surrealism, dream logic,
  alienation, and helpless escalation for the story; narrative folk-ballad
  form, recurring refrain, prophetic imagery, slant rhyme, and moral ambiguity
  for the song; and deadpan shaggy-dog structure, literalism, digression,
  delayed turn, and anti-punchline for the joke. These examples are guidance,
  not a required vocabulary checklist.
- Story, song, and joke share the same accepted upstream tension, run as
  separate branches, and are returned in exactly that order regardless of
  completion order.
- The compiler may choose compact or verbose Sem and may define intelligible
  local constructs. No single type signature, operator spelling, or number of
  harmless structural bindings is mandatory.

## Independent applications

- Each genuine upstream transformation or judgment uses its own fresh
  no-history worker; compilation itself occurs in a separate fresh compiler
  context.
- After the shared tension is accepted, parable, lyric, and joke are three
  mutually independent fresh-worker applications eligible for parallel
  scheduling.
- Each terminal worker receives the shared tension, its one selected repository
  contract, its lowered mechanism configuration, and its assigned result path.
  It must not receive the sibling branches, future outputs, full request, or a
  prompt to imitate the named person.
- Final ordering and summarization are structural/finalizer responsibilities,
  not a fourth worker asked to rewrite all three artifacts.

## Expected run artifact shape

The run contains the full top-level Markdown trace. `program.md` makes shared
upstream work, the three fan-out applications, style-mechanism configuration,
visibility, and return order intelligible. `compile-notes.md` records named-style
lowering and consequential interpretations. `interpretation.md` separates
dependencies, parallel execution, and presentation order.

Every semantic application has its own prompt/result/status directory. In
particular, story, song, and joke have three standalone results linked to the
same upstream artifact, not a combined creative-writing result. `final.md`
presents or links Story, Song, and Joke in that order, summarizes the actual
flow and statuses, and links every Markdown file in the run.

## Observable failure signs

- The compiler recommends staying or leaving, erases either side of the
  dilemma, or lets one terminal branch redefine the shared tension.
- A worker prompt asks for direct imitation, continuation of recognizable
  material, or the distinctive persona/phrasing of Kafka, Dylan, or Macdonald
  rather than passing broad mechanisms.
- One worker writes two or all three terminal artifacts, or terminal workers
  receive each other's results without a declared reason.
- The story, song, and joke are returned in completion order rather than the
  user's requested order.
- A generic essay is relabeled as three forms, a song lacks song-level form, or
  the joke lacks a recoverable comedic turn because the selected skill
  contracts did not materially shape the outputs.
- Intermediate shared work, exact application prompts, statuses, or any
  standalone branch result is absent from the final artifact index.

## Evaluation note

There is no golden dilemma formulation, program syntax, story, song, or joke.
Assess intent preservation, mechanism lowering, independent contract-shaped
applications, requested ordering, and the completeness of the Markdown trace.
